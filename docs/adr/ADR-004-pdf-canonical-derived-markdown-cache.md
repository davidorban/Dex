# ADR-004: Source of truth — PDF canonical, derived markdown cache under `.derived/`

- **Status**: proposed
- **Date**: 2026-05-16
- **Deciders**: David Orban
- **Tags**: librarian, storage, source-of-truth

## Context

The Librarian needs searchable text. PDFs are not directly indexable by QMD (which is a markdown-first search engine over the vault). Two options:

1. **PDF canonical, markdown derived** — keep PDFs as authoritative artefacts; extract them once to markdown into a `.derived/` cache; QMD indexes the cache. PDFs are version-controlled separately (or not at all — see gitignore status); the cache is reproducible from PDFs at any time.
2. **Markdown canonical, PDFs derivative or discarded** — extract once, then treat the markdown as the truth. PDFs become optional / archival. Saves storage but loses the original artefact, page-level visual fidelity, and the ability to cite to a viewable physical document.

The use case is **citation in deliverables aimed at other humans** (deal memos, ELT decks, briefs). Citing to a markdown extraction is unprofessional — readers expect a real PDF when they click the link. Throwing away the PDFs would also lose the User-Guide-defined `Subject/Geography/Document` directory structure that gives semantic meaning to file paths.

## Decision

**PDF is canonical.** The derived markdown lives under `06-Resources/Research/.derived/<relative-pdf-path>.md` and is treated as a reproducible cache.

The cache is:
- Gitignored (alongside the PDFs themselves under `06-Resources/Research/`).
- Regenerated by `librarian ingest` on demand. Idempotent on file sha1, so reruns only re-extract changed/new PDFs.
- Never edited by hand. Any manual edit will be wiped on the next ingest.

Each derived `.md` includes frontmatter with: source PDF path, subject category, geography, page count, sha1, ingestion date.

## Consequences

### Positive
- Original artefacts preserved. Citations point at viewable PDFs.
- Cache is recoverable from scratch — losing `.derived/` is annoying, not catastrophic.
- Sha1-based idempotency makes incremental ingest cheap.
- Frontmatter carries category/geography metadata into QMD, enabling future filtered queries (e.g., "search only Cryptocurrency/Regional Middle East").

### Negative
- Storage cost ~doubled vs. markdown-only (PDFs are typically much larger than their text extractions, but extraction does add ~5–20% on top of the PDFs).
- Two artefacts to keep in sync; programmer discipline (no manual `.derived/` edits) must be enforced by convention, not the filesystem.
- Initial ingest of 1,377 PDFs will be slow (probably 30–60 minutes wall time depending on PDF complexity); needs progress reporting and resume-on-failure.

### Neutral
- Decision is reversible at moderate cost: switching to "markdown canonical" would mean re-organising the cache, updating skill output to reference `.md`, and accepting the loss of PDF citations.
